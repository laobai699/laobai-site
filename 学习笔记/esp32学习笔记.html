🛠️ ESP32 & ESPHome 学习笔记

从零基础到自动化控制系统的进阶指南

一、 核心概念：硬件与灵魂

  • ESP32 芯片： 核心大脑，自带 Wi-Fi 和蓝牙。
  • DevKit (开发板)： 适合 DIY 的成品板，带 USB 接口，插线即用。
  • 固件 (Firmware)： 芯片内部的程序。刷入固件 = 给芯片安装大脑。

二、 软件生产线：ESPHome

ESPHome 是运行在电脑上的固件加工厂，通过 Python 驱动。

组件 角色 类比
YAML 配置文件 设计图纸：用文字描述需求。
Python 核心引擎 工头：负责看懂图纸并指挥编译。
C++ 底层语言 建筑材料：保证芯片运行极速。

三、 写入（刷机）流程

  1. 连接： 用 USB 线将 ESP32 连上电脑。
  2. 编写： 在 ESPHome 中写好 YAML（包含 Wi-Fi 信息和引脚定义）。
  3. 编译： 点击 Install，软件自动调用 Python 生成 .bin 固件。
  4. 烧录： 软件通过串口将固件“存入”芯片闪存。

四、 远程控制逻辑

  • 局域网： 芯片连接家里 2.4GHz Wi-Fi，获取内网 IP。
  • 异地控制： 需要一个“中间人”（如 Home Assistant 或云平台）。
  • 安全建议： 严禁映射端口，推荐使用加密隧道（如 Tailscale）。

五、 ⚠️ 避坑指南

1. 电压： 芯片核心为 3.3V，严禁直接接高压！

2. Wi-Fi： 不支持 5GHz，必须确保路由器开启 2.4GHz 频段。

3. 更新： 第一次刷机用 USB，之后修改只需点击 OTA 无线更新。
