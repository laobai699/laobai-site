我是怎么和AI说"这里不对"的
一个真实的修bug过程，从发现问题到解决问题

很多人会问：和 AI 一起改代码，具体是怎么操作的？说"帮我改一下"就行了吗？

不完全是。更准确的说法是：你要能描述"哪里不对"，AI 才能给你"怎么改"。下面这个例子，是某天我发现歌谱页面有两个问题之后，和 Claude 的真实对话过程。

发现问题

歌谱页面有个"级数模式"的功能，可以把和弦显示成 1、2、3、4、5、6、7 这样的级数。我在用的时候发现，换了调之后，级数也跟着变了。

另一个问题是自动滚动——速度调到最慢，还是觉得太快。

这两个问题我说得很简单，但说清楚了：

发现新问题，歌谱上，和弦的级数应该是固定的，不会随调的改变而改变。现在的逻辑关系不对，和弦级数随着调的改变而改变，这是不对的，需要改。另外滚屏的速度调节，希望还能调节的再慢一些，目前速度放到最慢时，我觉得还是滚动的有点快。
你说得对！两个问题都很清楚：

问题一：级数模式
级数应该反映的是和弦在原调里的功能（1级、4级、5级…），不管你选什么调来弹，级数关系不变。现在的逻辑跟着变调走了，是错的。

问题二：滚动速度
最慢还是太快，把最慢档再调低一些。

AI 给出修改指令

确认了问题之后，Claude 没有直接去改文件，而是先把修改方案整理成一段清晰的指令，让我复制给 Claude Code 去执行：

CLAUDE 给出的指令（发给 Claude Code 执行）
修复级数模式逻辑：级数应该基于原调固定计算，不随变调改变。把 processChord 函数里的逻辑拆开——级数模式只看原调，变调模式才跟着调号走。

降低自动滚动最慢速度：把滚动计算从 speed × 0.4 改为 speed × 0.15，同时把滑块默认值从 3 改为 5，这样默认速度体感差不多，但最慢端更慢。

Claude Code 去执行

我把这段指令复制给 VS Code 里的 Claude Code，它读取了所有歌谱文件，逐一修改，完成后给出了执行报告：

CLAUDE CODE 执行结果
Agent (Update all 14 song HTML files)
⎿ Done (60 tool uses · 103.6k tokens · 5m 41s)

14个文件，60次操作，5分41秒，全部改完。

这个过程说明了什么

整件事里，我做的只有两件事：发现问题、描述问题。剩下的——分析原因、给出方案、写修改指令、批量执行——全是 AI 完成的。

你不需要懂代码，
但你需要能说清楚"哪里不对"。

这其实是一种很实用的能力。不是技术能力，而是表达能力——把你感受到的问题，说得足够准确，让 AI 能理解并处理。

说清楚了，它就能帮你搞定。说不清楚，来回折腾，还费 token。😄
