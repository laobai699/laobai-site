怎样和AI交流修改代码
一个真实的例子，从发现问题到解决问题

很多人会问：和 AI 一起改代码，具体是怎么交流的？

与其解释，不如直接看一个真实的例子。下面是某天我和 Claude 的一段对话，完整还原了从发现问题到解决问题的过程。

那天我注意到歌谱页面有两个问题。

第一个问题：级数模式不对

歌谱上，和弦的级数应该是固定的，不会随调的改变而改变。现在的逻辑关系不对，和弦级数随着调的改变而改变，这是不对的，需要改。
你说得对！级数应该反映的是和弦在原调里的功能（1级、4级、5级…），不管你选什么调来弹，级数关系不变。现在的逻辑跟着变调走了，是错的。

需要把 processChord 函数的逻辑拆开：级数模式时，固定用原调计算；变调模式时，才跟着调号走。

注意这里的交流方式：我不需要懂代码，只需要描述"哪里的效果不对"。AI 来判断是什么原因，然后给出解决方案。

第二个问题：滚动太快

另外滚屏的速度调节，希望还能调节的再慢一些，目前速度放到最慢时，我觉得还是滚动的有点快。
明白了！把滚动计算的系数从 0.4 调小到 0.15，同时把默认值从最小调到中间，这样最慢端会慢很多，但默认速度体感和之前差不多。

然后呢？

两个问题说清楚之后，Claude 给出了一段完整的 Claude Code 指令，大概长这样：

请修改 songs 文件夹里所有歌谱文件，修复以下两个问题： 1. 修复级数模式逻辑 把 processChord 函数改为：级数模式时固定 用原调计算，变调模式时才跟调号走。 2. 降低自动滚动最慢速度 把滚动系数从 0.4 改为 0.15， 默认值从 3 改为 5。

我把这段指令复制粘贴到 VS Code 的终端里，发给 Claude Code，它自动读取了全部14个歌谱文件，逐一修改，几分钟就全部完成了。

整个流程是这样的

发现问题
›
用自己的话描述
›
AI分析原因
›
AI给出指令
›
复制给Claude Code执行
›
验证效果
不需要懂代码，
只需要能描述"哪里不对"。

这就是人和 AI 协同修改代码的真实样子。说清楚问题，剩下的交给它。
